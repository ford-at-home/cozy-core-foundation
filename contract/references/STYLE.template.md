# Voice: <voice-name>

> A voice file is your writing identity as code. Save filled-in voices to
> `~/.me/voices/<voice-name>.md` (or `<repo>/.me/voices/<voice-name>.md` for
> a per-repo override) and reference them by name in your `brief.md`.
>
> See `examples/voices/ford.md` (in this package) for a real-world worked
> example of the level of specificity that makes a voice file useful.
